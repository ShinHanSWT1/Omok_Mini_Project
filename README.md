# 🕹️ 오목 웹소켓 미니 프로젝트

> JSP / Servlet 기반 WebSocket 오목 게임  
> 실시간 양방향 통신을 이용한 2인 대전 오목 미니 프로젝트

---

## 📖 프로젝트 소개

본 프로젝트는 **JSP & Servlet 환경에서 WebSocket을 활용하여**
브라우저 간 **실시간 오목 게임**을 구현한 미니 프로젝트입니다.

HTTP의 단방향 통신 한계를 극복하기 위해  
**WebSocket을 이용한 실시간 통신 구조**를 학습·적용하는 것을 목표로 합니다.

## 🛠 기술 스택

### Backend
- Java (JDK 8+)
- JSP / Servlet
- WebSocket (javax.websocket)
- Apache Tomcat 9

### Frontend
- HTML / CSS
- JavaScript (WebSocket API)

---

## ⚙️ 주요 기능

- 🔗 WebSocket 기반 실시간 오목 대전
- 👥 2인 플레이어 매칭
- ⏱️ 턴 기반 돌 두기
- ❌ 자신의 턴이 아닐 경우 돌 두기 제한
- 🏆 승리 조건 판별 (5목 완성)
- 🔄 실시간 게임 상태 동기화

---

## 🧩 시스템 구조

- **JSP** : 화면(View) 구성
- **Servlet** : 초기 요청 처리
- **WebSocket ServerEndpoint** :  
  실시간 메시지 송수신 및 게임 로직 처리

---

## 🔄 실행 흐름

1. 사용자가 JSP 페이지 접속
2. WebSocket 서버 연결
3. 플레이어 입장 처리
4. 턴 정보 전달
5. 돌 두기 이벤트 전송
6. 서버에서 게임 상태 판단
7. 모든 클라이언트에 상태 브로드캐스트
